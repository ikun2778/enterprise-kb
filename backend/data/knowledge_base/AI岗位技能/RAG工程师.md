# RAG工程师技能指南

## 什么是RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的AI技术架构。它通过从外部知识库中检索相关信息，将其作为上下文传递给大语言模型，从而生成更准确、更有依据的回答。

RAG的核心思想是：先检索，后生成。相比直接让LLM回答问题，RAG能够利用最新的、特定领域的知识，有效减少幻觉问题。

## 核心技术栈

### 向量数据库

向量数据库是RAG系统的基础组件，用于存储和检索文档的向量表示。

常见选择：
- FAISS：Facebook开源的向量检索库，适合本地部署，性能优秀
- Milvus：分布式向量数据库，适合大规模生产环境
- Chroma：轻量级向量数据库，适合原型开发
- Pinecone：托管式向量数据库服务，无需运维

### Embedding模型

Embedding模型将文本转换为向量表示，是RAG质量的关键因素。

常用模型：
- BGE系列（BAAI/bge-large-zh-v1.5）：中文效果优秀
- M3E系列：支持中英文
- OpenAI text-embedding-3：效果好但需要API调用
- Sentence-Transformers：开源模型生态丰富

### 文档分块策略

合理的文档分块对检索质量至关重要：

- 固定大小分块：按字符数或token数切分，简单但可能破坏语义
- 递归分块：按层级分隔符递归切分，保持语义完整性
- 语义分块：基于语义相似度切分，效果最好但计算开销大
- Markdown感知分块：按标题层级切分，适合结构化文档

## 混合检索策略

### 向量检索

基于语义相似度的检索，能够理解同义词和语义关系。适合处理自然语言查询。

### BM25关键词检索

基于词频的传统检索方法，对精确匹配效果好。适合处理包含专有名词的查询。

### RRF混合重排

RRF（Reciprocal Rank Fusion）将多种检索结果融合排序：

```
RRF_score = Σ 1/(k + rank_i)
```

其中k通常取60，rank_i是文档在第i个检索器中的排名。

## 父子文档检索

父子文档检索（Small-to-Large）是一种优化策略：

1. 将文档切分为小块用于检索（提高精度）
2. 检索到小块后，返回其所属的父文档作为上下文（提高完整性）

这种方法兼顾了检索精度和上下文完整性。

## 评估指标

RAG系统的核心评估指标：

- 召回率（Recall）：检索到的相关文档比例
- 精确率（Precision）：检索结果中相关文档的比例
- MRR（Mean Reciprocal Rank）：正确答案排名的倒数均值
- 答案准确率：最终生成答案的正确性
- 幻觉率：生成内容中无依据信息的比例

## 常见优化方向

### 检索优化

- 查询重写：将用户问题改写为更适合检索的形式
- 查询扩展：生成多个相关查询提高召回
- 重排序：使用Cross-Encoder对初步结果精排
- 元数据过滤：利用文档元数据缩小检索范围

### 生成优化

- Prompt工程：设计有效的提示模板
- 上下文压缩：去除无关信息，保留关键内容
- 引用标注：让模型标注信息来源
- 流式输出：提升用户体验

### 工程优化

- 索引缓存：缓存常用查询结果
- 异步处理：并行执行检索和生成
- 分片存储：大规模文档的分布式存储
- 增量更新：支持知识库的实时更新
