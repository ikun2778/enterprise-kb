# LLM应用开发工程师

## 岗位职责

- 负责基于大语言模型的应用产品开发，包括对话系统、内容生成、智能助手等
- 设计和实现Prompt Engineering方案，优化模型输出质量
- 开发和维护LLM应用的基础架构，包括模型调用、上下文管理、输出解析等
- 负责RAG系统的开发和优化，提升知识问答的准确性
- 参与Agent系统的开发，实现多工具协同和任务编排
- 进行模型效果评估和A/B测试，持续优化产品体验

## 技术要求

- 精通Python编程，具备良好的软件工程实践
- 熟悉主流LLM（GPT、Claude、文心、通义千问等）的API调用和特性
- 精通Prompt Engineering，有丰富的提示词优化经验
- 熟悉LangChain等LLM应用开发框架
- 了解RAG架构，有知识库问答系统开发经验
- 熟悉FastAPI/Flask等Web框架
- 了解模型微调（LoRA、P-Tuning等）的基本流程
- 有产品思维，能够将业务需求转化为技术方案

## 常见技能

- Python
- LangChain / LlamaIndex
- Prompt Engineering
- RAG
- Agent / Tool Calling
- FastAPI
- OpenAI API / 各大模型API
- 向量数据库
- 模型微调 (LoRA/P-Tuning)
- 评估指标设计

## 面试重点

- Prompt Engineering的最佳实践和技巧
- LLM应用中的常见问题（幻觉、上下文长度限制等）及解决方案
- RAG系统的设计和优化经验
- Agent系统的架构设计（ReAct、Plan-and-Execute等）
- 如何评估LLM应用的效果
- 模型选型的考量因素
- 上下文窗口管理和优化
- 流式输出的实现方案
- 安全性考虑（内容审核、注入防护等）

## 项目要求

- 有完整的LLM应用开发和上线经验
- 有Prompt Engineering的系统性实践经验
- 有RAG或Agent系统的开发经验
- 能够独立完成从需求分析到产品上线的全流程

## 初级岗位能力标准

- 能够使用LangChain构建基础的LLM应用
- 掌握基本的Prompt Engineering技巧
- 能够实现简单的RAG系统
- 理解LLM的局限性和应对方案
- 能够使用FastAPI构建API服务
