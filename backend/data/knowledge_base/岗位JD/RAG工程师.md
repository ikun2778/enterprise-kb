# RAG工程师

## 岗位职责

- 负责企业级RAG系统的设计、开发和优化
- 设计和实现文档处理管线，支持多种格式（PDF/Word/HTML/Markdown）的解析和分块
- 优化检索策略，包括向量检索、关键词检索、混合检索和重排序
- 设计和实现Embedding模型的选型、微调和部署方案
- 负责向量数据库的选型、部署和性能优化
- 建立RAG系统的评估体系，持续优化检索质量和生成效果

## 技术要求

- 精通Python编程，3年以上相关开发经验
- 深入理解RAG架构，有大规模RAG系统开发经验
- 熟悉FAISS、Milvus、Pinecone等向量数据库的原理和使用
- 熟悉LangChain、LlamaIndex等RAG开发框架
- 了解Embedding模型原理，有模型微调经验者优先
- 熟悉BM25、TF-IDF等传统检索算法
- 了解重排序模型（Cross-Encoder）的使用
- 有分布式系统开发经验者优先

## 常见技能

- Python
- LangChain / LlamaIndex
- FAISS / Milvus / Pinecone
- HuggingFace Transformers
- BM25 / TF-IDF
- RRF (Reciprocal Rank Fusion)
- Cross-Encoder重排序
- 文档分块策略
- Embedding模型微调
- Docker / Kubernetes

## 面试重点

- RAG系统的完整架构设计
- 各种检索策略的对比和选择（向量vs BM25 vs 混合）
- 父子文档检索的实现原理
- RRF重排序算法的数学原理
- 文档分块策略的选择和优化
- Embedding模型的选型和评估
- RAG系统的评估指标（召回率、精确率、MRR）
- 如何减少RAG系统的幻觉问题
- 大规模向量数据库的部署和优化

## 项目要求

- 有完整的RAG系统开发和部署经验
- 处理过万级以上文档的RAG系统
- 有检索效果优化的实际经验（如RRF调参、重排序等）
- 有Embedding模型微调或评估经验者优先

## 初级岗位能力标准

- 能够独立设计和实现完整的RAG管线
- 熟悉至少一种向量数据库的使用和优化
- 能够进行检索效果的评估和优化
- 理解各种分块策略的优缺点
- 能够处理多种格式的文档解析
