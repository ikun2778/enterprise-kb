# 大模型应用工程师

## 岗位职责

- 负责大语言模型在企业场景中的应用落地，包括智能客服、知识管理、内容生成等
- 设计和实现基于LLM的应用架构，包括RAG、Agent、Fine-tuning等技术方案
- 负责Prompt Engineering和模型输出质量优化
- 开发模型服务化接口，支持高并发和低延迟的在线调用
- 建立模型评估体系，进行效果评估和持续优化
- 跟踪业界最新技术动态，探索前沿技术在业务中的应用

## 技术要求

- 精通Python编程，具备扎实的机器学习基础
- 深入理解Transformer架构和大语言模型原理
- 熟悉主流LLM（GPT、Claude、LLaMA、ChatGLM等）的特性和使用
- 精通LangChain/LlamaIndex等LLM应用开发框架
- 熟悉RAG系统的设计和优化
- 了解模型微调技术（LoRA、QLoRA、P-Tuning等）
- 熟悉向量数据库和检索技术
- 有模型部署和推理优化经验者优先
- 了解模型安全和对齐技术

## 常见技能

- Python
- PyTorch / TensorFlow
- LangChain / LlamaIndex
- Transformer架构
- RAG系统设计
- Agent / Tool Calling
- 模型微调 (LoRA/QLoRA/P-Tuning)
- 向量数据库 (FAISS/Milvus)
- Prompt Engineering
- 模型评估
- vLLM / TGI (推理优化)
- Docker / Kubernetes

## 面试重点

- Transformer架构的核心原理
- 大语言模型的训练和推理流程
- RAG系统的设计和优化方案
- Agent系统的架构设计（ReAct、Function Calling等）
- Prompt Engineering的方法论和实践经验
- 模型微调的方法和适用场景
- 模型评估的方法和指标
- 如何平衡模型效果和推理成本
- LLM应用的安全性问题和防护方案
- 前沿技术的了解和思考（MoE、长上下文等）

## 项目要求

- 有完整的LLM应用开发和部署经验
- 有RAG或Agent系统的生产级开发经验
- 有模型微调或评估经验
- 能够独立完成从技术选型到产品落地的全流程
- 有技术博客或开源贡献者优先

## 初级岗位能力标准

- 理解Transformer和LLM的基本原理
- 能够使用LangChain构建LLM应用
- 能够设计和实现基础的RAG系统
- 掌握Prompt Engineering的基本技巧
- 能够进行模型API调用和基础评估
