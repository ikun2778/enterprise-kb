# RAG面试题

## 基础概念

### 什么是RAG？相比微调有什么优势？

RAG（Retrieval-Augmented Generation）通过检索外部知识来增强LLM的生成能力。

相比微调的优势：
- 知识更新：RAG可以实时更新知识，微调需要重新训练
- 成本更低：无需GPU训练，只需维护知识库
- 可解释性：可以追溯信息来源
- 减少幻觉：基于检索的事实回答

### RAG系统的核心组件

1. 文档加载器：加载各种格式的文档
2. 文本分割器：将文档切分为适当大小的块
3. Embedding模型：将文本转换为向量
4. 向量数据库：存储和检索向量
5. 检索器：执行相似性搜索
6. LLM：基于检索结果生成回答

### 什么是Embedding？如何选择Embedding模型？

Embedding是将文本映射到高维向量空间的技术，语义相似的文本在向量空间中距离更近。

选择标准：
- 语言支持：是否支持目标语言
- 维度大小：高维精度高但计算慢
- 推理速度：影响系统响应时间
- 模型大小：影响部署成本

## 检索策略

### 向量检索和关键词检索的区别

向量检索：
- 基于语义相似度
- 理解同义词和近义词
- 对自然语言查询友好
- 计算开销较大

关键词检索（BM25）：
- 基于词频和文档频率
- 精确匹配效果好
- 对专有名词友好
- 计算速度快

### 什么是RRF？如何实现？

RRF（Reciprocal Rank Fusion）是一种融合多个检索结果的方法：

```
RRF_score(d) = Σ 1/(k + rank_i(d))
```

其中k是常数（通常60），rank_i(d)是文档d在第i个检索器中的排名。

实现步骤：
1. 对每个检索器的结果按相关性排序
2. 计算每个文档在每个检索器中的排名
3. 使用RRF公式计算融合分数
4. 按融合分数重新排序

### 什么是父子文档检索？

父子文档检索（Small-to-Large）是一种优化策略：

1. 将文档切分为小块（子文档）
2. 检索时使用小块提高精度
3. 返回结果时使用父文档提供完整上下文

优势：
- 检索精度高：小块语义更集中
- 上下文完整：父文档包含完整信息
- 减少噪音：避免不相关内容干扰

## 优化技巧

### 如何处理文档分块？

分块策略：
- 固定大小：简单但可能破坏语义
- 递归分割：按层级分隔符切分
- 语义分割：基于语义相似度
- Markdown感知：按标题层级切分

最佳实践：
- 块大小：通常200-1000 tokens
- 重叠：10-20%的重叠
- 元数据：保留文档来源信息

### 如何减少RAG系统的幻觉？

策略：
- 提高检索质量：确保检索到相关内容
- 引用来源：让模型标注信息来源
- 置信度评估：对回答进行置信度评分
- 事实验证：对关键信息进行二次验证
- 限制生成：要求模型只基于检索内容回答

### 如何评估RAG系统？

评估维度：
- 检索质量：召回率、精确率、MRR
- 生成质量：准确性、相关性、流畅性
- 端到端：答案正确率、幻觉率

评估方法：
- 人工评估：最准确但成本高
- 自动评估：使用标准数据集
- A/B测试：对比不同方案

## 高级话题

### 如何处理多模态数据？

方案：
- 图像描述：使用视觉模型生成描述
- 表格提取：使用专门的表格解析工具
- PDF解析：使用PyMuPDF等工具
- 统一表示：将多模态数据转换为文本

### 如何实现增量更新？

方案：
- 文档版本管理：跟踪文档变更
- 增量索引：只更新变化的文档
- 缓存失效：更新时清除相关缓存
- 原子更新：保证更新的一致性

### 如何优化大规模检索？

策略：
- 分片存储：将索引分布到多个节点
- 缓存机制：缓存常用查询结果
- 预计算：提前计算常见查询
- 索引优化：选择合适的索引类型
